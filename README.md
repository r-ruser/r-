# PubMed 文献抓取可视化工具

一个基于 **R Shiny** 的 PubMed 文献抓取界面工具。  
适合不想手动改脚本、希望通过图形界面完成检索、摘要抓取、JCR/IF 匹配和 Excel 导出的用户。

## 功能概览

- 图形界面输入检索参数，无需手改脚本
- 支持输入关键词或直接输入自定义 PubMed 检索式
- 支持设置起始年份、结束年份、抓取条数、文章类型、语言等条件
- 可选抓取摘要
- 可选匹配 **JCR 分区** 与 **影响因子 IF**
- 支持指定默认保存文件夹，自动导出 `.xlsx`
- 提供运行日志框，实时显示检索进度
- 提供停止按钮，可在任务执行过程中发出停止请求
- 后台执行任务，界面不会因为长时间抓取而假死

## 界面包含的主要参数

### 基础检索参数

- **关键词**：支持换行、英文逗号、中文逗号、分号分隔
- **关键词连接方式**：`AND` 或 `OR`
- **自定义 PubMed 检索式**：填写后优先使用，不再自动拼接关键词
- **起始年份 / 结束年份**
- **文章类型**：如 `Journal Article`、`Review`、`Clinical Trial`
- **语言**：不限、英文、中文、其他
- **抓取条数**
- **单次请求超时秒数**

### 结果增强参数

- **同时抓取摘要**：逐篇访问 PubMed 页面提取摘要，速度相对更慢
- **匹配 JCR / IF**：需要额外上传对照文件
  - JCR 对照文件必须包含列：`ISO`、`JCR`
  - IF 文件必须包含列：`JCRAbbreviation`、`JIF`

### 导出参数

- **默认保存文件夹**：结果完成后自动保存到该目录
- **导出文件名**：默认导出为 `.xlsx`

## 运行环境

- R >= 4.1 建议
- 推荐使用 RStudio 运行

## 安装依赖

```r
install.packages(c(
  "shiny", "rentrez", "openxlsx", "dplyr", "rvest",
  "readr", "readxl", "writexl", "httr", "xml2"
))
```

## 运行方式

将项目文件下载到本地后，在 R 或 RStudio 中运行：

```r
shiny::runApp("app_pubmed_gui_pro.R")
```

如果你把文件放在某个项目目录中，也可以先切换工作目录后再运行。

## 推荐的首次测试方式

建议先用小规模参数确认环境正常，再逐步增加功能：

1. 关键词填写：`tuberculosis`
2. 起始年份：`2023`
3. 结束年份：`2025`
4. 抓取条数：`5`
5. 先不要勾选“同时抓取摘要”
6. 先不要勾选“匹配 JCR / IF”
7. 点击“开始抓取”

如果这一步可以正常运行，再继续测试摘要抓取和 JCR/IF 匹配。

## 输出结果说明

程序默认导出 Excel 文件，通常包含：

- `PMID`
- `标题`
- `摘要`
- `ISO`
- `发表年份`
- `类型`
- `影响因子`
- 如果启用匹配，还会包含 `JCR` 和 `IF` 等信息

同时会附带一张查询信息表，用于记录本次检索式。

## 日志与停止机制说明

### 运行日志框

日志框会持续显示程序执行到了哪一步，例如：

- 正在检索 PubMed
- 检索完成，命中多少条 PMID
- 正在读取概要
- 正在抓取摘要
- 正在匹配 JCR / IF
- 正在保存结果

### 停止按钮

停止按钮采用的是**协作式停止**，不是强制杀进程。

这意味着：

- 它不会粗暴中断当前这一条网络请求
- 但会在当前步骤结束后尽快停止
- 在逐条抓摘要时，通常会在当前 PMID 处理结束后停止

这种方式更稳，不容易把 R 会话直接打坏。

## 常见问题

### 1. 点击开始抓取后看起来没反应

优先检查以下几点：

- 是否已经正确安装依赖包
- 是否网络可访问 PubMed
- 抓取条数是否设得太大
- 是否勾选了“同时抓取摘要”，因为该步骤本身较慢
- 查看日志框和状态框是否已有输出

### 2. 结果没有自动保存

通常是保存目录没有写权限。建议将默认保存目录改为你本地有权限的文件夹，例如桌面或某个项目目录。

### 3. JCR / IF 匹配失败

请检查上传文件的列名是否符合要求：

- JCR 文件：`ISO`、`JCR`
- IF 文件：`JCRAbbreviation`、`JIF`

列名不一致时会报错。

### 4. 抓摘要很慢

这是正常现象。摘要抓取需要逐篇请求 PubMed 页面，比只抓 PMID 和标题慢很多。建议先小批量测试，再扩大范围。
